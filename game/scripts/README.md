# Game Scripts

- `player/` contains player control and replication-ready state code.
- `npc/` contains interactable NPC foundations, names, dialogue triggers, and future schedule hooks.
- `dialogue/` contains JSON-driven NPC dialogue runtime code and UI glue.
- `inventory/` contains stackable inventory state with save/load serialization.
- `reputation/` contains Fringe, Gray Line, and Solenne reputation tracks.
- `systems/` contains cross-cutting runtime systems such as saving.
- `quest/` contains data-driven quest state management and quest UI.
- `demo/` contains first playable prototype orchestration.
